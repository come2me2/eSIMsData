## Следующие версии

При создании новой версии:
1. Обновить версионирование файлов (дата/время в query параметрах)
2. Обновить `CACHE_VERSION` в `data-loader.js` при необходимости
3. Создать новый тег: `git tag -a v1.0.1 -m "Описание изменений"`
4. Обновить этот файл
